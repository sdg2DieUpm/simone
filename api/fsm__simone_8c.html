◆ check_playback_over()

static bool check_playback_over ( fsm_t *  p_this)
static

Check if the playback of the sequence is over and thus the player can start inputting the sequence.

You can solve this problem in different ways. One possible solution is to check if the playback index has reached a special value, for example 0xFF, which is a very large value that will be really difficult to set as game property (normal sequences will be much shorter).

Other ways to solve it are also valid. You can use global flags, or other mechanisms.

Additionally, you must check that the final OFF-timer (the last gap) has expired.

Create a new Simone FSM.

This FSM implements a system that represents an adaptation of the game Simon. The system is activated by pressing a button. The system is deactivated by pressing the same button for the same amount time.

The basic implementation of this FSM assumes that the player plays with a matrix keyboard and an RGB light. The system can add more keyboards and RGB LEDs to make a version of the game.

When the system is IDLE it does, the display is IDLE. The system is in a low power mode.

To better understand the system, please watch the following video:

This is the HW diagram of the system:

This function initializes the default values of the FSM struct and calls to the port to initialize the HWs associated to the devices.
